# 设计文档（2 页）

> 作品：**DecisionMate** —— 会替你拿主意、且服务整个"局"的本地活动规划执行 Agent
> 一句话：同一道题站高一层——不只答"怎么把行程排好、执行掉"，而答"**一个 Agent 到底该替你做多少决定**"。

## 0. 核心洞察

赛题埋着一个内在矛盾：一边要"一键帮你做完"（别让我动手），一边用"递给老婆/发朋友看看"暗示"别自作主张"（我要审）。全自动违背第二条、事事都问违背第一条。**唯一同时满足两条的，是按"置信度 × 代价"决定自动到什么程度**——这就是本作品的大脑。"递给别人看"还藏着第二个信号：**决策不是一个人做的**——于是升维为**群体决策协调器**，与主轴是同一个引擎。

---

## 1. Planning 策略

一句话目标走一条**手搓的轻量决策循环**（不用 LangChain/标准 ReAct，避免框死自定义置信度逻辑）。整条链：

```
目标 →[意图解析] 列"已知/未知"清单（绝不默默填默认值）
     →[置信度×代价] 对每个待定决定评估 → 推导 做/建议/问
     →[内部冲突协调] 在"无唯一正确答案"处带理由权衡（娃要儿童餐 vs 老婆要低卡）
     →[动线优化] 没点名菜系就把吃饭锚到"要去的活动"附近；活动带"饭前/饭后"时序
     → 第一版 Plan（每个决定带 confidence / basis / cost / reasoning / disposition）
     → 你回答 ask 类 + 对 suggest 类拍板（ask 给预设选项，也可自己填）
     →[协同评审] 多人各自改 → 自动合并无争议 → 撞车按 weight 给带理由建议 → 你裁决
     →[继续加要求] 自然语言再生成（/api/refine：改活动/菜系/距离/预算/过敏/辣度，真用高德重搜重排）
     → 看"定稿"满意 → 一键执行 + 异常兜底 → send_plan + 可分享方案卡
```

**决策大脑（`rules.py`）**：`disposition = f(confidence, cost)`

| 条件 | 处置 |
|---|---|
| `cost==high` 或 `confidence<0.4` | **ask**（停下来问） |
| `confidence≥0.7` 且 `cost==low` | **auto**（直接做） |
| 其余 | **suggest**（给建议） |

- **置信度是算出来、可指读的**：餐厅按"命中几条已知约束（儿童餐/低卡/距离/过敏/预算）"加减得出；固定决策也用 `confidence_from_factors` 给出算式（如"基准0.50 −0.15(预算未知) −0.05(问错代价高)=0.30"），把**缺失项**显式压低置信度——"先承认不确定"的认知起点。
- **置信度会更新**：低置信决定被反馈确认/推翻后重算，`confidence_basis` 记下"被谁修正"。
- **自主性可调**：保守/平衡/大胆三档平移阈值（高代价仍兜底问，安全底线不丢）。

**协同评审（`review.py`）**：无争议改动自动吸收（"海鲜过敏"→纳入硬约束并剔除海鲜店）；多人改到同一决定**拎成 Conflict**，按 `采纳置信度 = 0.55×内容契合 + 0.45×作者weight` 给"采纳谁+为什么"的建议，拍板权留给你（老婆改的天然比不熟朋友更可信）。真实分享链接 `/join`，朋友用**自己手机**提交改动。

**信任账户（脑洞主线）**：你每确认它一次信任就长一点；用熟后对"预算/到家时间"会**按历史默认填好、但仍可改**（不锁死，最终决定权在你）。

**情感辅线是主轴的特例**：发现"老婆减肥"→ 推断送高热量蛋糕适得其反 → 礼物建议成"花+展览票"，但 `disposition=suggest`（情绪敏感的事置信度到不了"直接做"）。一个引擎，主辅一体。

**AI 层（可选增强，`llm.py`/`llm_tools.py`/`amap.py`）**：意图解析与候选生成可切真实 LLM + 高德真实 POI（按你的真实定位搜附近、画真实路线图）。**纪律：AI 只负责"听懂+找店"，`置信度×代价→做/建议/问` 永远由确定性大脑算**——主轴不进黑盒、可解释、可复现。调不通自动回退本地 Mock 库，带磁盘缓存（命中秒回、可离线复演），**现场不赌网络**。

## 2. 工具调用链路（`tools.py`，全 Mock，可制造缺失/失败/延迟）

覆盖"**查—订—买—送—发**"五类（贴合美团能力），返回数据**带齐决策字段**（人均/儿童友好/低卡/过敏原/距离/排队/等位时长/坐标），大脑靠这些算置信度：

| 阶段 | 工具 |
|---|---|
| 查 | `search_restaurants`（带约束过滤）、`search_activities` |
| 订 | `book_restaurant`、`book_activity`、`join_queue` |
| 买 | `order_flowers`、`order_cake`、`buy_ticket` |
| 送 | `arrange_delivery` |
| 发 | `send_plan` → 生成移动端**可分享方案卡**（`card.py`） |

**执行按依赖顺序**：预订活动（多活动逐个）→ 订餐厅 → 礼物（花/票）→ 发计划。每笔成功下单累加 **GMV 计数器**（屏幕实时跳动）；事务用 `_commit`/`rollback` 记账，保证 GMV 与账本一致。Mock 用固定随机种子 + `forced` 强制项，演示**可复现**，并能主动制造"满了/超时/失败"。

**商业价值（连带率，单次会话可数）**：一句话撬动"活动门票 + 餐厅 + 礼物"多个 SKU；组局（≥3 人）客单天然高于个人。`_business_summary` 当场算出"连带品类数 / 单局客单 / vs 单点外卖的倍数"——这些是美团天天在测的真实指标，不是拍脑袋。护城河 = 数据规模 × 履约网络，只有美团生态给得齐。

## 3. 异常处理机制（多数队伍的死穴 = 我们的得分点）

执行是**带状态的流程**（待执行/执行中/成功/失败），异常全做且能演：

1. **餐厅满了** → 自动 `join_queue` 报等位 ETA；ETA 超阈值（30min）→ 从满足约束的候选里**自动改订备选**，并**重建时间线**让方案与实际订到的店一致。
2. **工具超时** → 仅对 `timeout` 类**自动重试**（最多 2 次）。
3. **中途某步失败 → 事务完整性**：把"送花"视为一笔事务 `[下花单 → 送达]`，第二步失败时**回滚第一步花单**（退款、GMV 回退），给出**补偿**（到店现买，不影响惊喜）；独立的展览票事务不受牵连。
4. **意图模糊** → 反问澄清（ask 类决定即此机制）；活动线上订不到 → 兜底"到场购票，不影响行程"。
5. **外部依赖降级**：高德/LLM 调不通 → 自动回退本地 Mock 库；定位失败 → 可手动输入地点（全国 POI 搜索，不锁城市）；浏览器无定位权限 → 手动地址兜底。顶层 `try` 安全网保证执行不崩。

> "顺风顺水能跑"是门槛，"出事了还能优雅稳住"才出彩。

## 4. 工程纪律 & 自检

- **守灵魂字段**：`confidence / confidence_basis / cost / reasoning / disposition` 任何时候不为"先跑通"而删（`selftest.py` 自动守护）。
- **认知主轴不进黑盒**：AI 只管"听懂+找店"，做/建议/问永远由确定性大脑算。
- **现场不赌网络**：真实数据带磁盘缓存、可离线复演。
- **双重达标**：`python3 -m agent.selftest` 内部逻辑 **93/93** ✅ ·  `python3 check_acceptance.py` PRD 9 条验收瞬间 **9/9** ✅。
