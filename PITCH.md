# 路演手册（评委话术 + 演示脚本 + 防翻车）

> 给现场用的"刀鞘"。工程已经做完，这页决定评委记住的是"一个会判断该替你决定多少的 Agent"，还是"又一个吐行程的聊天机器人"。

---

## 0. 30 秒电梯陈述（开场必背）

> "市面上的 Agent 要么傻乎乎全自动、让你不放心，要么什么都来问你、等于没帮上忙。我们做的是**第三种**：它对每个决定先算两件事——**我有多大把握、错了代价多大**——有把握又便宜的**直接做**，拿不准或代价高的**停下来问**，情感敏感的**给建议让你拍板**。而且它服务的不是一个人，是**整个局**：方案发给老婆/朋友，每个人都能改，它自动合并、在分歧处给带理由的建议。一句话进去，一桌子事办完——还把手机递给老婆的那一屏也做好了。"

---

## 0.5 「AI 在哪？」—— 一句话答死（最高频技术质疑，开场后第一道防线）

**Q：你这决策脑是规则不是 AI，在 AI 黑客松里 AI 含量够吗？**

> "分两层看：**AI 负责'想得到'，决策脑负责'敢不敢替你定'。**
> 听懂你一句口语化的自然语言、按目标**实时生成**逼真候选、应对我们没预设过的约束（你说'去上网'它就找网咖，你说'蹦床'它就搜蹦床馆）——这些全是 LLM 干的，是 AI 做到了人做不到的'秒级理解 + 生成'。
> 但'**要不要替你刷这张卡**'，我们**故意不交给 LLM 拍脑袋**，留在一套可解释、可复现的安全层里。"

**记死这个类比（脑洞奖最强的人话钩子）：**

> "它像一个**真正懂你的朋友**——帮你把位订了、把娃的避辣也考虑了，但在**刷你信用卡之前，一定先发消息问你**。
> 全自动的 Agent 不懂分寸、让你不放心；什么都问的 Agent 帮不上忙。我们做的是'**懂分寸**'——这恰恰是赛题'**把手机递给老婆看看**'里埋的那句潜台词：别自作主张。"

> 一句拔高：这是 **Human-in-the-loop Agent 里"自主性边界"问题的工程化答案**——别人用"不确定就问"的拍脑袋启发式，我们把它做成了有明确输入输出、有降级策略的可计算函数。

## 1. 两句话差异化（最高频、最致命的问题）

**Q：你这跟美团现在的 AI 推荐 / 跟 ChatGPT 排行程有什么本质区别？**

> "推荐解决的是'给你看什么'，我们解决的是'替你决定到什么程度、并把它做掉'。**核心不是推荐算法，是一条自动化边界规则**：`置信度 × 代价 → 做/建议/问`——这是把'Agent 该不该替你拍板'这个问题**形式化成了一个可计算、可解释的函数**，而不是一个会聊天的推荐位。ChatGPT 给你一段文字，我们给你一个**带署名分歧协调、能下单、出事能回滚**的执行闭环。"

记住三个词钉死它：**边界规则 / 群体协调 / 执行闭环**。

---

## 2. confidence 怎么来的（技术派评委必问）

**Q：你的 confidence 是怎么算的？LLM 拍的吗？可靠吗？**

> "**不是让 LLM 拍脑袋**。我们故意把决策权留在一套**确定性规则**里：每个候选按它命中了几条已知约束（儿童餐 / 低卡 / 距离 / 过敏原 / 预算 / 口味）加减得分，**缺失项（如预算未知）显式压低置信度**。在协同评审里还多一个输入——**是谁说的**（`author.weight`，老婆的权重高于不熟的朋友）。LLM 只用来'听懂需求 + 生成候选'，**打分和定 做/建议/问 全是规则**——所以它可解释、可复现，不是黑盒。第一版规则先粗后细，这正是 PRD 要求的路径。"

加一句拔高（有 AI 背景评委会买账）：
> "这本质上是 **Human-in-the-loop Agent** 里'自主性边界'问题的一个工程化回答——大多数作品用'不确定就问'的启发式，我们把它做成了有明确输入输出和降级策略的函数。"

## 3. 演示脚本（5–7 分钟，按这个顺序演，别即兴）

**先讲'这是什么'（30s 电梯陈述）→ 再演，全程念出'置信度/处置'。**

1. **【主演示·走本地库，稳】** 跑家庭场景：一句话进去 → 念出"已知/未知清单"（先承认不确定）。
   - 指着 **孩子常识=✅直接定**（高置信低代价）、**预算/到家=❓停下来问**（低置信高代价）、**午睡=💡给建议**——"看，它知道哪些该自己定、哪些不敢替你定。"
2. **内部冲突**：娃要儿童餐 vs 老婆要低卡 → 找到两全的店，念出取舍理由。
3. **情感高光**：发现老婆减肥 → 主动建议蛋糕换花+票，"**是建议、不是擅改**"。
4. **协同评审（重点演，升维亮点）**：`as 老婆` 换近的店、`as 朋友甲` 换海鲜火锅、`as 朋友乙` 加"海鲜过敏" → **自动合并过敏约束、餐厅撞车给带 weight 的建议** → A 裁决 → v2。
5. **执行 + 兜底**：一键执行 → **故意演满座→换备选、超时→重试、配送失败→回滚补偿**，GMV 计数器跳动。
6. **递给老婆那一屏**：点"📲 打开可分享方案卡"——"这就是小明把手机递给老婆看的样子"，手机端卡片 + 一键复制发小张。
7. **（加分·可选）** 切到"🧠 用 AI 实时生成"，输一句全新的"带孩子去海洋馆再吃日料" → 展示候选是 AI 现场生成的真实店名，"**同一套大脑、候选不再写死**"。AI 慢就先说"它在实时查"，有缓存秒回兜底。

**收尾一句**："一句话撬动多笔交易、激活下午消费低谷、组局客单更高——这不是搜索推，是把事情做完。"

## 5. 现场翻车应对（提前练）

- **LLM 超时/限流**：别慌、别重试到尴尬。直接说："它在实时查询，我们先看缓存结果——**注意主路径是本地确定性引擎，AI 只是增强**"，然后继续走本地库版本。降级是**设计好的**，大方承认。
- **网络全断**：整场走本地库 + 缓存，9 条验收瞬间一条不少。
- **被问倒**：承认 + 给方向。"这条我们现在是 Mock/规则，下一步会接 X"——评委吃**诚实 + 清晰的演进路径**，不吃硬撑。

## 8. 第二轮评审后的补丁（务必读）

### 8.1 真实性怎么讲：把"地点真实性"和"支付 Mock"分开（最重要的一条防守）

评委最致命的一招是掏手机搜你推荐的店。所以话术要**主动分两层**：

> "我们把'**地点必须真**'和'**支付走 Mock**'分开：①**地点**——已接入**高德真实 POI 实时搜索**，候选是真实店名/地址/人均/评分/离家距离（围绕'家'做周边搜索，距离也是真的），**您现在就能在地图上搜到**；②**支付/履约**——这一层是赛题官方明确要求的 Mock。③高德调不通时自动回退 AI 实时生成 / 本地真实连锁库，永不崩。"

**演示纪律**：主线开「🌐 用真实数据（高德POI）」，候选页会标「候选来自高德真实 POI（可搜证）」——**主动请评委拿手机搜任意一家**，搜得到=当场加分。绝不把编的店当真实数据讲。

### 8.2 增量 GMV：给一条"可被追问、可被证伪"的假设链（不是三句口号）

评委问"凭什么说是增量"，按这个**透明假设模型**答（数字是**占位假设**，标清楚、可替换成真实数据）：

```
下午 3–6 点本地消费时段分布（示意，非真实数据）：
 上午 ▇▇▇▇▇
 午高峰 ▇▇▇▇▇▇▇▇▇
 下午低谷 ▇▇            ← Agent 把"帮我安排"做成确定性入口，填这块
 晚高峰 ▇▇▇▇▇▇▇▇▇▇

增量假设链（每一步都欢迎评委挑战）：
 设 一次成功组局 触发 SKU 数 = N（餐厅+门票+礼物+用车，演示里实测约 3）
 设 这些里"用户本来不会下"的发现型订单占比 = p（待真实A/B验证）
 设 决策成本从 ~15 分钟压到 ~1 分钟 → 放弃率下降 → 转化率 +w 个百分点
 => 单次会话增量 GMV ≈ 客单 × p × (1+w)；组局(≥3人) 客单天然高于个人
```

关键是**承认它是模型**："这些系数现在是假设，**接入真实数据后第一件事就是用 A/B 把 p 和 w 测出来**——我要给您的不是一个拍脑袋的数字，是一个**能被证伪的增量机制**。" 这句比报一个假数字强一百倍。

> ⚠️ 别编造并声称是美团真实财报数字。要么标"示意/假设"，要么引用你真查到并能给出处的公开数据。诚实的假设链 > 不可信的精确数字。

### 8.3 多人协同：演成"否决 → 重规划 → 推送新卡"的 30 秒来回（别演成独角戏）

评委容易把"视角切换"看成一个人自言自语。所以这一段要演出**协商张力**：

> `as 老婆` 把餐厅改近 + 加"海鲜过敏" → 屏幕上 Agent **自动合并约束、剔除海鲜店、餐厅撞车给带 weight 的建议** → A 裁决 → **重新生成 v2 方案，并重新生成那张可分享卡片推回去**。

一句话钉死差异点："**我们做的不是个性化搜索，是家庭/小团体决策的协商层**——别人改、它合、撞车它给带理由的建议、最后人拍板。这个赛道没人做。"

### 8.4 confidence 阈值被深问怎么答（技术派必问"0.7 哪来的"）

> "阈值现在是**规则先验 + 代价分级**定的：高代价（钱花出去难收回、影响整条时间线）一律压到'问'，这是**保守安全侧**的有意设计，不是调参调出来的精确值。它的价值不在'数值准'，在'**行为可解释、可复现**'——同样输入永远同样处置。**有了用户'采纳/拒绝'数据，第一步就是把每个决定的拒绝率回灌、在线校准这些阈值**——每一次拍板都是一条监督信号。"

> 一句收尾：把它定位成 **"AI 决策的安全层"**——LLM 负责听懂和找候选，这层负责'该不该替你拍板'，**故意不让 LLM 进这个决策黑盒**，这正是认知向的核心。
