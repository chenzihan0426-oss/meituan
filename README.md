# 会替你拿主意的本地活动规划执行 Agent

> 美团 AI Hackathon · 赛题：本地场景短时活动规划与执行 Agent
> 一句话：同一道题，我们站高一层——不答"怎么把行程排好、执行掉"，而答"**一个 Agent 到底该替你做多少决定**"。

一句话目标（"今天下午带老婆孩子出去玩，别太远，老婆在减肥"）进去 → 一份**带置信度、带理由、能一键执行**的方案出来，还能**发链接让朋友各自上去改**。

---

## 它到底特别在哪

市面上的"规划 Agent"要么**全自动**（事事替你做主，容易踩雷），要么**事事都问**（等于没帮你）。这个赛题本身就埋了矛盾：一边要"一键帮我做完"，一边用"递给老婆/发朋友看看"暗示"别自作主张"。

**我们的答案：让 Agent 对每个决定先自问两件事，再决定自动到什么程度。**

| 动手前自问 | |
|---|---|
| 1️⃣ 我有多确定你会认可？ | **置信度** |
| 2️⃣ 如果我错了，代价多大？ | **代价** |

```
置信度高 且 代价低   → ✅ 直接做      （避辣、避酒吧街这种育儿常识，自己定）
中等                → 💡 给建议       （留不留午睡口子，给方案你点头）
置信度低 或 代价高   → ❓ 停下来问     （预算、几点到家，不敢替你拍板）
```

这台"决策大脑"是**确定性规则**（不进黑盒、可解释、可复现）：餐厅按"命中了几条已知约束"算置信度，预算/过敏/距离/口味都是输入。**AI 只负责"听懂你说什么 + 找真实的店"，要不要替你定永远由这台大脑算。**

而且它服务的不是一个人，而是整个**"局"**：方案出来后进入**协同评审**——你把链接发给老婆/朋友，大家各自上去改，Agent 自动合并无争议的、撞车处按「内容契合度 × 谁说的(权重)」给带理由的建议、把拍板权留给你。**个人助理 → 群体决策协调器，同一个大脑。**

## 一次完整流程会看到什么

```
说一句目标
  └→ 解析意图，列出【已知 / 未知】清单（绝不默默填默认值）
     └→ 对每个决定算 置信度×代价 → 做 / 建议 / 问
        └→ 内部冲突协调（娃要儿童餐 vs 老婆要低卡 → 找两全的店，讲清取舍）
           └→ 你回答"问"类（预算/到家时间）→ Agent 据此重新规划餐厅与行程
              └→ 发链接给朋友 → 大家各自改 → 自动合并 + 撞车建议 → 你裁决 → v2
                 └→ 一键执行（查→订→买→送→发）+ 异常兜底 + GMV 计数跳动
```

## 主要能力

- 🧠 **置信度×代价决策大脑**：每个决定都带 `置信度 / 依据 / 代价 / 理由 / 处置`，确定性、可解释。
- 👨‍👩‍👧 **群体协同（真实分享链接）**：生成链接，朋友用**自己手机**打开看方案、提交"换餐厅/加约束"，你这边自动收取并入评审。
- 📍 **真实数据 + 自动定位**：接高德 POI，按你的**真实位置**搜附近（浏览器 GPS → 自动转高德坐标），也可手动改地点。候选都是评委能当场搜证的真实店。
- 🗺️ **听得懂各种活动**：游乐场/博物馆/海洋馆/公园/电影/**网吧**/**KTV**/逛街，认不出的（蹦床/密室…）就**按你的原话搜附近**，绝不默默给你换成别的。
- ⏱️ **智能时间线**：按你定的"几点到家"**倒排**整条行程（"晚上12点"=午夜，不会变成中午）。
- 💰 **内嵌商业价值**：一句话撬动门票+餐厅+礼物多笔交易，GMV 计数器实时跳动。
- 🛡️ **异常兜底**：餐厅满了→排队/换备选；工具超时→重试；下单失败→**回滚同笔事务**并补偿。
- 💝 **情感体贴**：发现"老婆在减肥"→ 主动把蛋糕建议换成"花+展览票"（建议，非擅改）。

## 快速开始（零依赖，只需 Python 3.9+）

```bash
# 自动演示·家庭主场景（推荐先看这个）
python3 run_demo.py
python3 run_demo.py friends          # 朋友局泛化（2男2女、口味不一，同一套代码）

# 真人交互 CLI（用 `as 老婆` 切视角去改方案）
python3 -m agent.cli

# 图形化 WebUI（本地网页，零依赖）：一句话 → 方案确认 → 协同反馈 → 定稿 → 执行 + 朋友协同分享
python3 -m agent.web                  # 浏览器开 http://127.0.0.1:8000

# 自检（守灵魂字段 + 主轴规则 + 验收瞬间，93 项断言）
python3 -m agent.selftest
# 验收自检（对照 PRD 9 条「必须演出来的瞬间」核验真实输出）
python3 check_acceptance.py
```

> **全程只用 Python 标准库**，不联网也能跑（高德/LLM 是可选增强，调不通自动回退本地 Mock 库，带磁盘缓存可离线复演）。

### 开启真实数据（可选）

复制 `agent/llm_config.example.json` 为 `agent/llm_config.json`，填入你自己的高德 key（和可选的 LLM key）即可。**仓库里不含任何密钥**。

## 代码结构

```
agent/
  models.py        # 核心数据结构（Decision/Plan/Edit/Conflict…）含 5 个"灵魂字段"
  rules.py         # 置信度×代价→处置；餐厅打分；把"谁改的"纳入置信度
  intent.py        # 意图解析 + 缺失识别（产出"已知/未知"清单）+ 活动/口味识别
  tools.py         # 工具层（查订买送发），全 Mock，可制造缺失/失败/延迟
  amap.py          # 高德真实 POI + 自动定位（WGS84→GCJ02 坐标转换、逆/正地理编码）
  planner.py       # 生成第一版 Plan + 内部冲突协调 + 时间线倒排 + GMV 估算
  review.py        # 协同评审回合：自动合并 + 冲突建议 + 再规划
  execution.py     # 执行 + 异常兜底（满了/超时重试/失败回滚）+ GMV 计数
  orchestrator.py  # 端到端编排（被 demo / cli / web 复用）
  web.py           # 图形化 WebUI + 朋友协同分享页（标准库 http.server，零依赖）
  card.py          # "递给老婆/发小张"的移动端可分享方案卡（自包含 HTML）
  llm.py/llm_tools.py # 可选 AI 层：意图解析 + 候选生成（OpenAI 兼容，带缓存+回退）
  demo.py / cli.py # 自动演示 / 真人交互入口
check_acceptance.py  # 命令行验收自检（对照 9 条瞬间）
DESIGN.md            # ≤2 页设计文档
```

## 工程纪律

- **守灵魂字段**：`confidence / confidence_basis / cost / reasoning / disposition` 任何时候不为"先跑通"而删（`selftest` 自动守护）。
- **认知主轴不进黑盒**：AI 只管"听懂 + 找店"，`置信度×代价→做/建议/问` 永远由确定性大脑算，可解释、可复现。
- **现场不赌网络**：真实数据带磁盘缓存，首次联网后即固定，可离线复演。
