# 知道"该替你决定多少"、且服务整个"局"的本地活动规划执行 Agent

> 美团 AI Hackathon · 赛题：本地场景短时活动规划与执行 Agent
> 主攻**技术认知向**，辅以**情感向**。

这**不是**一个"会规划、会执行"的 Agent（那是门槛），而是一个**知道"自己该替你决定多少"**的 Agent。
它对每一个决定，动手前先自问两件事：

> 1. 我有多确定用户会认可？（**置信度**）
> 2. 如果我错了，代价有多大？（**代价**）
>
> - 置信度高 **且** 代价低 → **直接做**（auto）
> - 中等 → **给建议，由你拍板**（suggest）
> - 置信度低 **或** 代价高 → **停下来问**（ask）

并且它服务的不是一个人，而是整个"局"：方案出来后进入**协同评审回合**——攒局人 A 分享给老婆/朋友，
大家各自上去改，Agent 自动合并无争议改动、在撞车处给**带理由的建议**、把拍板权留给 A。
这把产品从"个人助理"升维成**群体决策协调器**，而它和上面那条主轴是**同一个大脑**。

## 快速开始（零依赖，只需 Python 3.9+）

```bash
# 自动演示·家庭主场景（推荐先看这个）
python3 run_demo.py
#   等价：python3 -m agent.demo --scenario family

# 自动演示·朋友局泛化（2 男 2 女、口味不一，同一套代码）
python3 run_demo.py friends

# 真人交互（协同评审用 `as 老婆` 切视角去改方案）
python3 -m agent.cli

# 自检（守灵魂字段 + 主轴规则 + 验收瞬间，93 项断言）
python3 -m agent.selftest

# 验收自检（对照 PRD 9 条「必须演出来的瞬间」核验真实输出，命令行）
python3 check_acceptance.py

# 图形化验收台（本地网页，仅标准库零依赖；可切场景/切冲突裁决，左侧 9 条清单自动打勾）
python3 -m agent.web
```

> `agent.web` 不是 PRD 要交付的'产品前端'（PRD §2 明确不做花哨前端）——它是**给你自己验收**的一面镜子：
> 跑的是同一套真实引擎，把决策的置信度/处置、协同评审、执行兜底、GMV 计数器渲染成网页，
> 并把 9 条验收清单实时打勾。绝不搭真实多端/网络同步（仍遵守 §2/§7.3）。

> 全程**只用 Python 标准库**，不联网、不接真实支付/第三方 API（全 Mock），不依赖 LLM——
> 置信度第一版用清晰的规则函数实现，确保"真能跑"且可复现。

## 一次演示会看到的关键瞬间（对应 PRD 验收清单）

1. **高置信直接定**：孩子相关常识（避辣、避酒吧街、避免太累）Agent 自己定，并说明依据。
2. **低置信停下来问**：预算 / 到家时间，Agent 不敢替你定，停下来问。
3. **中置信给建议**：留不留午睡口子，给建议等你拍板。
4. **内部冲突协调**：娃要亲子+儿童餐 vs 老婆要清淡低卡 → 找到两全的店，讲清取舍。
5. **情感高光**：发现老婆减肥 → 主动建议把蛋糕换成"花 + 展览票"（建议，非擅改）。
6. **协同评审（升维亮点）**：老婆把餐厅改近一点、朋友甲改成海鲜火锅、朋友乙加"海鲜过敏"
   → Agent 自动合并过敏约束（并剔除海鲜店）、在餐厅撞车处给**带理由 + 含 weight** 的建议 → A 裁决 → 生成 v2。
7. **异常恢复**：餐厅"满了"→ 自动排队报 ETA / 等位过久换备选；工具超时 → 自动重试；
   配送失败 → 回滚同笔事务里已下的花单并补偿（事务完整性）。
8. **一键执行 + 状态可见 + GMV 计数器跳动**；最后 `send_plan` 把方案发出去。
9. **泛化**：`friends` 场景同一套代码切到 4 人朋友局也能跑。

## 代码结构

```
agent/
  models.py        # 核心数据结构（Decision/Plan/Edit/Conflict…）——含 5 个"灵魂字段"
  rules.py         # 置信度 × 代价 -> disposition；餐厅打分；把'谁改的'纳入置信度
  intent.py        # 意图解析 + 缺失识别（产出"已知/未知"清单）
  tools.py         # 工具层（查订买送发），全 Mock，可制造缺失/失败/延迟
  planner.py       # 生成第一版 Plan + 内部冲突协调 + GMV 估算
  review.py        # 协同评审回合：自动合并 + 冲突建议 + 再规划
  execution.py     # 执行 + 异常兜底（满了/超时重试/失败回滚）+ GMV 计数
  display.py       # 让"思考"可见（PRD §9 格式，ANSI）
  scenarios.py     # 家庭 / 朋友 两个场景预设
  orchestrator.py  # 端到端编排（被 demo / cli / web 复用）
  demo.py / cli.py # 自动演示 / 真人交互入口
  web.py           # 图形化验收台 + 交互向导（标准库 http.server，零依赖）
  card.py          # "递给老婆/发小张"的移动端可分享方案卡（自包含 HTML）
  llm.py / llm_tools.py # 可选 AI 层：意图解析 + 候选生成（OpenAI 兼容，带缓存+回退）
  acceptance_spec.py # 验收清单（9 条）单一事实来源，供 check_acceptance 与 web 共用
  selftest.py      # 内部逻辑自检（93 项）
check_acceptance.py  # 命令行验收自检（对照 9 条瞬间核验真实输出）
DESIGN.md          # ≤2 页设计文档（Planning / 工具链路 / 异常处理 / 增量 GMV）
PITCH.md           # 路演手册：评委话术 + 演示脚本 + 防翻车
```

详见 `DESIGN.md`。
