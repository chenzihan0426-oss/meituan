# DecisionMate · 作品介绍

> 美团 AI Hackathon · 赛题：本地场景短时活动规划与执行 Agent
> 一句话：同一道题站高一层——不只答"怎么把行程排好、执行掉"，而答"**一个 Agent 到底该替你做多少决定**"。

---

## 一、核心理念：做一个"拎得清"的 AI

### 1.1 我们读出的赛题矛盾

赛题里埋着一对内在矛盾：
- 一边要"**一键帮你做完**"——别让我自己动手；
- 一边用"**递给老婆 / 发朋友看看**"暗示"**别自作主张**"——这事我要审。

**全自动**违背第二条（事事替你拍板，容易踩雷）；**事事都问**违背第一条（等于没帮你）。市面上的规划 Agent 不是太莽就是太怂。

### 1.2 我们的答案

**唯一同时满足两条的，是按"置信度 × 代价"决定自动到什么程度。** Agent 对每个决定先自问两件事：

| 动手前自问 | |
|---|---|
| 1️⃣ 我有多确定你会认可？ | **置信度** |
| 2️⃣ 如果我错了，代价多大？ | **代价** |

```
置信度高 且 代价低   → ✅ 直接做   （避辣 / 避酒吧街这种育儿常识，自己定）
中等                → 💡 给建议    （留不留午睡口子，给方案你点头）
置信度低 或 代价高   → ❓ 停下来问  （预算、几点到家，不敢替你拍板）
```

这台"决策大脑"是**确定性规则、可解释、可复现**：餐厅按"命中了几条已知约束"算置信度，预算/过敏/距离/口味都是输入。**AI 只负责"听懂你说什么 + 找真实的店"，要不要替你定，永远由这台大脑算。**

> 一句话：**能办的直接办，拿不准的给建议，只有它真不知道的才停下来问你。绝不丢一堆链接让你自己挑。**

### 1.3 第二个洞察：决策不是一个人做的

"递给别人看"还藏着第二层——**决策是一个"局"的事**。于是我们把个人助理**升维为群体决策协调器**：方案发链接给同行的人，各自上手机改，Agent 自动合并无争议的、撞车处按「内容契合 × 谁说的(权重)」给带理由的建议、把拍板权留给你。**个人助理 → 群体协调器，同一个大脑。**

## 二、使用场景（一次完整旅程）

### 场景 A：家庭周末出游
输入一句话：**"今天下午带老婆孩子出去玩，别太远，老婆在减肥。"**

1. **意图解析**：列出【已知：有 5 岁孩子→避辣/避酒吧街、别太远→10km 内、老婆减肥→低卡】【未知：预算？几点到家？忌口？】——**绝不默默填默认值**。
2. **决策大脑逐项处置**：
   - 避辣/避酒吧街 → ✅ 直接定（育儿常识，置信度 0.92）
   - 选餐厅 → 内部冲突协调：娃要儿童餐 vs 老婆要低卡，**找两全的店**并讲清取舍
   - 预算、几点到家 → ❓ 停下来问（代价高，给你**预设选项**也可自己填）
   - 留不留午睡口子 → 💡 给建议
   - 送礼：发现"减肥"→ 把蛋糕**建议**换成"花+展览票"（情感体贴，仍是建议非擅改）
3. **协同**：发链接给老婆/朋友 → 他们改 → 自动合并 + 撞车建议 → 你裁决。
4. **定稿确认**：看融合后的新方案 + 时间线 + 出行提醒，满意才执行。
5. **一键执行**：查→订→买→送→发，GMV 实时跳动，异常自动兜底。

### 场景 B：朋友局（泛化，同一套代码）
**"周末四个朋友小聚，想吃顿好的，有人无辣不欢。"** → 自动认出 **4 人**、口味协调（保证有辣口菜、怕辣的分盘），不再出现"午睡口子/儿童餐"等家庭味。

### 场景 C：临时改主意（自然语言再生成）
- "**吹海风喝啤酒**" → 安排海边活动 + 啤酒馆/大排档；
- "**下午去游乐园，吃完饭再去唱K**" → 多活动并存，按 **饭前→吃饭→饭后** 排顺序；
- "**换便宜点的 / 我对花生过敏 / 想吃火锅**" → 真用高德重搜重排，循环改到满意为止。

## 三、功能全景（从理念到实现）

| 模块 | 做了什么 | 代码 |
|---|---|---|
| 🧠 **决策大脑** | 置信度×代价→做/建议/问；置信度是**可指读的算式**（如 基准0.50 −0.15(预算未知)=0.30）；自主性可调（保守/平衡/大胆） | `rules.py` |
| 🗣️ **意图解析** | 产出"已知/未知"清单；识别人数、活动、菜系、场景词（海边/啤酒）；**否定感知**（"不吃海鲜"≠想吃）；过敏设为硬约束 | `intent.py` |
| 🧩 **内部冲突协调** | 在"无唯一正确答案"处带理由权衡（娃 vs 老婆），写清为什么选它、为什么排除别的 | `planner.py` |
| 🗺️ **动线优化** | 没点名菜系就把吃饭锚到"要去的活动"附近（同商圈/综合体）；活动带**饭前/饭后**时序，多活动并存 | `planner.py` |
| 👨‍👩‍👧 **群体协同** | 真实分享链接 `/join`，朋友用自己手机改；自动合并无争议、按 weight 给撞车建议、你裁决 | `review.py`/`web.py` |
| 🔁 **自然语言再生成** | "加要求"→识别改活动/菜系/距离/预算/过敏/辣度→高德重搜重排；定稿可循环改到满意 | `/api/refine` |
| 🤝 **信任账户** | 你每确认一次信任就长；用熟后对"预算/到家时间"**按习惯默认填好、但仍可改**（不锁死） | `prefs.py` |
| 📍 **真实数据** | 高德真实 POI + 浏览器/手动**全国定位** + 真实路线图（随方案自动重画） | `amap.py` |
| 🛠️ **执行闭环** | 查→订→买→送→发，多活动逐个预订，GMV 实时计数，产出可分享方案卡 | `execution.py`/`card.py` |
| 🛡️ **异常兜底** | 满了→排队/换备选、超时→重试、失败→**回滚同笔事务**并补偿；外部依赖降级 | `execution.py` |
| 🤖 **可选 AI 层** | 意图解析/候选生成可切真实 LLM；调不通自动回退本地 Mock + 磁盘缓存（离线复演） | `llm.py` |

> **关键纪律**：AI 只管"听懂 + 找店"，`置信度×代价→做/建议/问` 永远由确定性大脑算——**认知主轴不进黑盒、可解释、可复现，现场不赌网络。**

## 四、创新性

1. **重定义这道题**：业界默认在比"谁排得更全/执行得更顺"，我们指出真问题是"**该替你决定多少**"，并用 `置信度×代价` 把它做成一台**可解释、可调**的决策大脑——这是认知层面的升维，不是功能堆砌。
2. **升维为群体决策协调器**：从"伺候一个人"到"协调一个局"，用的还是同一个引擎（采纳谁的改动 = 又一次 置信度×代价，且把"谁说的"纳入置信度）。
3. **补全用户没说的**：先承认"用户要什么本身是不确定的"，主动列"未知"、情感推断（减肥→换礼物）、否定感知（不吃海鲜≠想吃）——把"听懂言外之意"做成机制。
4. **可迭代的人机共识**：定稿确认 + 自然语言循环再生成 + 信任账户"默认但可改"——决定权始终在用户手里，体验符合真实使用逻辑。

---

## 五、完整性（一个真能跑通的闭环）

- **全链路闭环**：一句话 → 规划 → 协同评审 → 定稿 → 执行（查订买送发）→ 可分享方案卡，**端到端真能跑**。
- **三种入口**：命令行自动演示 / 真人交互 CLI / 图形化 WebUI（含朋友协同分享页），同一套编排复用。
- **双重自检**：内部逻辑 **93/93**（`agent.selftest`）+ PRD 9 条验收瞬间 **9/9**（`check_acceptance.py`），可当场核验。
- **工程纪律**：守 5 个"灵魂字段"（confidence/basis/cost/reasoning/disposition）不为"先跑通"删；真实数据带缓存可离线复演。

## 六、应用效果（出彩在"分寸"与"出事也稳"）

- **内部冲突协调**：娃要儿童餐 vs 老婆要低卡 → 找两全的店并讲清取舍。
- **多人分歧协调**：老婆要换最近、朋友要换更高档 → 按权重给"采纳谁+为什么"的建议。
- **该问才问**：常识自己定、预算/时间停下来问——不烦你、也不瞎来。
- **情感体贴**：减肥当口把蛋糕建议成花+票（建议非擅改）；带娃避辣避酒吧街。
- **异常优雅**：餐厅满了自动改订备选并重排时间线；送花失败回滚退款并建议到店现买。"顺风顺水能跑是门槛，出事还能稳住才出彩。"

## 七、商业价值（可统计、有护城河）

### 7.1 一句话撬动"连带"——单次会话就能数出来
一句话目标，撬动 **活动门票 + 餐厅 + 礼物** 多个 SKU。系统当场算出：

- **连带品类数**（一单几个 SKU）、**单局客单**、**相对单点外卖的倍数**——都是美团天天在测、可优化的**真实指标**，不是拍脑袋的假数（`_business_summary`）。

### 7.2 三条真实增量
1. **激活低谷时段**：家庭"周六下午"本是本地消费转化低谷（缺好入口）；Agent 把"帮我安排"变成一个**确定性消费入口**。
2. **发现型订单**：满座兜底 / 口味协调时推荐用户"本来不会发现"的店——**纯增量**。
3. **组局乘数**：组局（≥3 人）客单天然高于个人，一句话带动多笔交易。

### 7.3 护城河
把"懒得安排"的沉默、零散、偶发需求，转成可被美团履约网络承接的**结构性流量入口**。护城河 = **数据规模 × 履约网络**，只有美团生态能同时给齐——别的平台抄不走。

## 八、技术架构（一图看懂）

```
一句话目标
  │
  ▼
[intent] 意图解析（已知/未知、人数、活动/菜系/场景词、否定感知、过敏硬约束）
  │
  ▼
[planner] 第一版 Plan：每个决定带 置信度/依据/代价/理由/处置
  │  ├─ rules：置信度×代价 → 做/建议/问
  │  ├─ 内部冲突协调（娃 vs 老婆）
  │  └─ 动线优化（就近吃）+ 时间线倒排（饭前/饭后）
  ▼
[你] 回答"问"类（预设选项/自己填）+ 对"建议"拍板
  │
  ▼
[review] 协同评审：自动合并 + 带权重撞车建议 + 你裁决  ←→  /api/refine 自然语言再生成
  │                                                     （高德重搜重排，循环到满意）
  ▼
[定稿确认] 看新方案 + 出行提醒 → 满意才执行
  │
  ▼
[execution] 查→订→买→送→发 + 异常兜底（满座改订/超时重试/失败回滚）+ GMV 计数
  │
  ▼
[card] 可分享方案卡（递给老婆 / 发朋友）

可选增强：amap 真实 POI/定位/路线图 · llm 真实意图解析/候选生成（均失败回退本地 Mock + 缓存离线复演）
```

## 九、如何运行 / 验证

```bash
python3 run_demo.py                 # 命令行自动演示（家庭场景）
python3 run_demo.py friends         # 朋友局泛化
python3 -m agent.cli                # 真人交互 CLI（as 老婆 切视角改方案）
python3 -m agent.web                # 图形化 WebUI（浏览器开 127.0.0.1:8000）

python3 -m agent.selftest           # 内部逻辑自检 → 93/93 ✅
python3 check_acceptance.py         # PRD 9 条验收瞬间 → 9/9 ✅
```

> 全程仅用 Python 标准库，零第三方依赖；不联网也能跑。开启真实数据：复制 `agent/llm_config.example.json` 为 `agent/llm_config.json` 填入自己的高德/LLM key（仓库不含任何密钥）。
