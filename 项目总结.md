## 一、这是什么

输入一句话目标，输出一份**带置信度、带理由、能一键执行**的本地活动方案，并支持**把链接发给朋友各自上去改**。

```
"今天下午带老婆孩子出去玩，别太远，老婆在减肥"
        ↓
 一份排好的方案（活动+餐厅+礼物+时间线）
 每个决定都标注：我有多大把握、凭什么、要不要替你定
        ↓
 你回答几个关键问题 → 发给朋友改 → 一键执行下单
```

## 二、核心设计思想

赛题埋着一个矛盾：一边要"一键帮我做完"，一边用"递给老婆/发朋友看看"暗示"别自作主张"。
全自动违背第二条，事事都问违背第一条。

**唯一同时满足两条的，是按"置信度 × 代价"决定自动到什么程度。** 这就是整个项目的大脑：

| 动手前自问 | 推导 |
|---|---|
| 置信度高 **且** 代价低 | ✅ **直接做**（育儿常识：避辣、避酒吧街） |
| 中等 | 💡 **给建议**（留不留午睡口子，你点头） |
| 置信度低 **或** 代价高 | ❓ **停下来问**（预算、几点到家） |

两条关键纪律：
1. **决策大脑是确定性规则**，不进黑盒——可解释、可复现。餐厅按"命中几条已知约束"算置信度，预算/过敏/距离/口味都是输入。
2. **AI 只负责"听懂 + 找店"**，要不要替你定永远由这台大脑算。

并且它服务的不是一个人，而是整个**"局"**：升维成**群体决策协调器**，且与主轴是同一个引擎。

## 三、功能全景

### 决策与规划
- **置信度×代价决策**：每个决定带 5 个"灵魂字段"（`confidence / confidence_basis / cost / reasoning / disposition`）。
- **内部冲突协调**：娃要儿童餐 vs 老婆要低卡 → 找两全的店并讲清取舍。
- **回答后重新规划**：你回答预算后**重选餐厅**（超预算的掉出）；回答到家时间后**倒排整条时间线**。
- **情感辅线**：发现"老婆减肥"→ 把蛋糕建议换成"花+展览票"（建议，非擅改）。

### 真实数据与定位
- **高德真实 POI**：候选都是评委能当场搜证的真实店，带真实人均/距离/评分。
- **自动定位**：浏览器 GPS → 自动转高德坐标（WGS84→GCJ02，国内差几百米必须转）→ 搜你附近；也可手动输入地点。
- **听得懂活动**：游乐场/博物馆/海洋馆/公园/电影/**网吧**/**KTV**/逛街；认不出的（蹦床/密室…）按**原话搜附近**。
- **智能时间线**：按"几点到家"倒排（"晚上12点"=午夜，不会变中午）。

### 协同与执行
- **真实分享链接**：朋友用自己手机打开链接，看方案、提交"换餐厅/加约束"，你这边自动收取并入评审。
- **协同评审**：自动合并无争议的、撞车处按「内容契合度 × 谁说的(权重)」给建议、拍板权留给你。
- **闭环执行**：查→订→买→送→发，GMV 计数器实时跳动。
- **异常兜底**：满了→排队/换备选；超时→重试；失败→**回滚同笔事务**并补偿。

### 工程
- **零依赖**：全程只用 Python 标准库，不联网也能跑。
- **可离线复演**：高德/LLM 带磁盘缓存，首次联网后即固定。
- **自动回退**：高德/LLM 调不通自动回退本地 Mock 库。

## 四、技术架构

```
agent/
  models.py        核心数据结构（含 5 个"灵魂字段"）
  rules.py         置信度×代价→处置；餐厅打分；把"谁改的"纳入置信度
  intent.py        意图解析 + 缺失识别 + 活动/口味识别
  tools.py         工具层（查订买送发），全 Mock
  amap.py          高德真实 POI + 自动定位（坐标转换 / 逆正地理编码）
  planner.py       第一版 Plan + 冲突协调 + 时间线倒排 + GMV
  review.py        协同评审：自动合并 + 冲突建议 + 再规划
  execution.py     执行 + 异常兜底 + GMV 计数
  orchestrator.py  端到端编排（demo/cli/web 复用）
  web.py           图形化验收台 + 朋友协同分享页（http.server，零依赖）
  card.py          移动端可分享方案卡
  llm.py/llm_tools.py  可选 AI 层（OpenAI 兼容，带缓存+回退）
  demo.py / cli.py     自动演示 / 真人交互入口
```

三个入口（同一套引擎）：
- `python3 run_demo.py` —— 自动演示
- `python3 -m agent.cli` —— 真人交互（`as 老婆` 切视角）
- `python3 -m agent.web` —— 图形化验收台（http://127.0.0.1:8000）

## 六、已知限制与边界

- **协同走轮询**而非实时推送（每 4 秒），属有意取舍：简单稳、不怕现场翻车。这一条突破了原 DESIGN"不做多端同步"的纪律。
- **真实数据需高德/LLM key**：仓库不含密钥，照 `agent/llm_config.example.json` 自填；不填则走本地 Mock。
- **手机访问分享链接**需与电脑同一 WiFi；自动探测局域网 IP，必要时链接可手动改 IP。
- **活动/时间解析**基于规则+LLM，覆盖常见说法；极端口语可能需换个说法。
